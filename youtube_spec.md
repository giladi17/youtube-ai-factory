מסמך אפיון טכני וארכיטקטורה: YouTube AI Content Factory
1. מטרת המערכת ותקציר מנהלים (Executive Summary)
הסבת מערכת אלגוטריידינג מבוססת סוכנים לשרשרת ייצור דיגיטלית (Pipeline) אוטומטית המפיקה סרטוני יוטיוב. המערכת תפעל בארכיטקטורת Microservices מבוססת אירועים (Event-Driven), תוך שימוש בסוכני AI אוטונומיים (OpenClaw) הרצים על גבי תשתית קוברנטיס. המטרה היא אוטומציה מלאה של תחקיר, כתיבה, הפקת וידאו, עריכה והפצה (SEO), תוך מינימיזציה של עלויות מחשוב.

2. ארכיטקטורת תשתית וענן (Infrastructure as Code)
הפרויקט מבוסס על שכתוב קוד ה-Terraform הקיים:

VPC & EKS: שימור התשתית הרשתית ואשכול הקוברנטיס המנוהל.

Storage (Amazon S3): יצירת באקטים חדשים המיועדים למדיה בלבד (אחסון תסריטים, קבצי שמע, תמונות, קבצי וידאו גולמיים ו-MP4 סופיים).

ניהול הרשאות (IAM & Secrets): החלפת מפתחות ה-API הפיננסיים בניהול סודות מאובטח עבור YOUTUBE_API_KEY, HEYGEN_API_KEY (או שירות מקביל), ו-OPENAI_API_KEY.

ניהול עומסים ועלויות (Karpenter & FinOps): יישום Scale-to-Zero. שרתי EC2 עתירי ביצועים (למשל עם מעבדים חזקים לרינדור וידאו) יוקצו דינמית רק עם הפעלת סוכן העריכה, ויושמדו מיד בסיום התהליך. בנוסף, תוגדר מערכת Billing Alarms, וכל חיבור לדומיין או שירות חיצוני יוגדר ללא חידוש אוטומטי כדי להבטיח שלא נחויב שוב בשנה הבאה ללא בקרה.

3. מערך הסוכנים (OpenClaw Microservices)
הסבת 5 סוכני שוק ההון ל-4 סוכני מדיה ייעודיים שירוצו כ-Pods עצמאיים:

Agent 1: Scriptwriter (לשעבר סוכן איסוף נתונים): סריקת מקורות מידע ו-RSS טכנולוגיים, בחירת נושא, וכתיבת תסריט יוטיוב מותאם (Hook + Body + CTA). פלט: קובץ טקסט/JSON ב-S3.

Agent 2: Avatar Director (לשעבר אנליסט): האזנה ל-S3. בעת קבלת תסריט, התממשקות ל-API של שירות יצירת וידאו AI עם תמונת אווטאר קבועה ליצירת וידאו Lip-Sync. פלט: קובץ וידאו גולמי (Green Screen/Transparent) ב-S3.

Agent 3: Video Editor (לשעבר מבצע עסקאות): מופעל על גבי שרת חזק זמני. מריץ סקריפטים של FFmpeg לחיבור וידאו האווטאר עם B-roll, כתוביות אוטומטיות ומוזיקת רקע. פלט: קובץ MP4 סופי ומוכן לשידור ב-S3.

Agent 4: SEO & Publisher (לשעבר מדווח): ניתוח התסריט להפקת כותרת Clickbait אופטימלית, תיאור ותגיות. התממשקות ל-YouTube Data API להעלאת הסרטון ופרסומו. פלט: קישור חי ליוטיוב.

4. זרימת עבודה מבוססת אירועים (Event-Driven Workflow)
Trigger: הפעלה יזומה דרך CLI/Webhook או מתוזמנת דרך Kubernetes CronJob.

Phase 1: יצירת K8s Job עבור Agent 1 -> כתיבת תסריט -> שמירה ב-S3 -> סיום Job.

Phase 2: זיהוי אובייקט חדש ב-S3 מפעיל את Agent 2 -> הפקת וידאו מול API חיצוני -> העלאת חומר גלם ל-S3 -> סיום Job.

Phase 3: הקצאת משאבים דינמית (Karpenter) והפעלת Agent 3 -> רינדור מקומי -> שמירת סרטון מוגמר -> סיום Job והשמדת שרת ה-EC2.

Phase 4: הפעלת Agent 4 -> העלאה ליוטיוב -> שליחת חיווי הצלחה.

5. שלבי ביצוע (Roadmap)
Milestone 1: Terraform Refactoring - התאמת התשתיות, ה-S3, וההרשאות.

Milestone 2: Kubernetes Configurations - עדכון ה-Jobs וכללי ה-Karpenter למבנה הדו-שלבי (סוכנים קלים מול סוכן עריכה כבד).

Milestone 3: OpenClaw Agents Logic - פיתוח והסבת קוד הפייתון/JS עבור מיומנויות הסוכנים החדשות.

Milestone 4: E2E Pipeline Testing - הרצת טסט מקצה לקצה ויצירת סרטון דמו ראשון בערוץ.
